Phase 1: Accuracy and integrity baseline (critical foundation)
1.1 Define a guidance trust model in map flow: heading trust, location trust, route trust, and overall guidance mode.
1.2 Fix alignment semantics so status labels and turn instructions cannot conflict.
1.3 Add configurable event-level discovery radius with a strict fallback default.
1.4 Add client-side anti-cheat plausibility checks: unrealistic speed, jump distance, stale fix age, duplicate submit window.
1.5 Show trust warnings in non-blocking UX for temporary uncertainty.

Phase 2: Logging enforcement and anti-cheat hardening (depends on Phase 1)
2.1 Gate logging by event radius and trust policy.
2.2 Extend discovery payload contract with integrity metadata snapshot.
2.3 Add duplicate and replay protections in submit flow (lock, cooldown, idempotency strategy).
2.4 Define backend validation contract for distance and plausibility checks with backward compatibility.

Phase 3: Guidance quality and sensor resilience (depends on Phase 1)
3.1 Improve heading smoothing and threshold tuning for stability plus responsiveness.
3.2 Add GPS-course fallback when magnetometer quality is low.
3.3 Add non-blocking calibration assistance with auto-recovery.
3.4 Add distance trend hints to reinforce confidence.

Phase 4: Route guidance integration (depends on Phase 1, parallel with late Phase 2)
4.1 Add provider-agnostic route service abstraction with timeout, cancellation, and throttling.
4.2 Add route normalization (geometry, ETA, distance, next maneuver).
4.3 Add route lifecycle hook with meaningful-movement refresh policy.
4.4 Render route polyline and mode indicator while preserving compass fallback.

Phase 5: UX clarity and actionability (depends on Phases 2-4)
5.1 Show required radius and current distance in logging UI.
5.2 Add explicit mode badges: Route Active, Compass Only, GPS Fallback, Sensor Limited.
5.3 Ensure warnings are actionable and concise.
5.4 Improve accessibility labels for directional and integrity states.

Phase 6: Performance and operational safety (depends on integrated Phases 2-5)
6.1 Memoize expensive derived values and reduce effect-chain churn.
6.2 Add refresh guards and request dedupe for route updates.
6.3 Add telemetry hooks for suspicious events and degraded guidance.

Phase 7: Verification and rollout
7.1 Add unit tests for turn wrapping, thresholds, trust scoring, and plausibility checks.
7.2 Add integration tests for event radius gating, anti-cheat blocking, and guidance mode transitions.
7.3 Add field tests for noisy sensors, urban turns, route loss, and spoof-like movement patterns.
7.4 Roll out in staged flags: accuracy first, anti-cheat gating second, route third.
